# 第1部　マーケティング / コンテンツ

## 0. 市場・競合の前提（調査サマリ）
- メンズエステ店舗は全国1万店超。多くが紙・Excel・LINE運用。
- 競合（tasteck/Caskan/Shop Manage 等）は「集客・予約」「顧客管理」に軸足。**委託金（バック）計算まで踏み込む製品は少ない（推測）**。
- 価格水準は月額0〜30,000円、特化型で5,000円/月〜が一相場。
- **示唆**：ShiftLinkは「委託金（バック）自動計算」「オーナー×セラピスト双方の業務オペ」「インストール不要PWA」で差別化。特にカード手数料・10%上乗せ自動化は紙・Excel運用の最大の苦痛＝締め日の計算地獄を直撃。

## 1. ターゲット / ペルソナ
- **メイン**：個人〜5名規模のメンズエステ店舗オーナー（スマホ中心・ITは苦手寄り・経営兼プレイヤー）。
- **ペルソナA（オーナー）**：月末の委託金計算に2〜3時間、カード手数料/上乗せの計算ミスでトラブル、シフトがLINEで埋もれる。→ 自動化・一元化・簡単さを欲する。
- **ペルソナB（セラピスト）**：自分のバックが不透明、シフト申請が流れる、深夜の不安。→ 透明な売上確認・簡単申請・緊急SOSの安心。
- オーナーが導入主体、**セラピストの使いやすさが定着の鍵**。

## 2. キャッチコピー
- コア：**「シフトも予約も委託金も、スマホひとつで終わる。」**
- メイン（FV）：**「締め日の電卓、もう閉じていい。」** ／ サブ「シフト・予約・委託金まで、スマホひとつで回るメンズエステ管理アプリ」「インストール不要。今日から14日間、無料で。」
- サブ案：A「月末の計算地獄を、ワンタップで。」／B「LINEで埋もれるシフトと予約を、ひとつの画面に。」／C「ITが苦手でも大丈夫。スマホが使えれば、もう使えます。」

## 3. 差別化3本柱（LPで明示）
1. **委託金（バック）計算がここまで自動**：カード手数料(6%+80円)・カード10%上乗せ・日別/月次集計・印刷/PDF出力まで完結。
2. **インストール不要のPWA × 高級感デザイン**：DL不要、ホーム画面に置くだけ。白×オレンジの上質UI。
3. **オーナーとセラピスト両方が使う業務アプリ**：シフト承認・ルーム自動振り分け・本日ダッシュボード・緊急SOS・出退勤打刻・プッシュ通知。
> 比較表は他社名を出さず「紙/Excel/LINE運用」との比較に留める（他社を直接批判しない）。

## 4. LPセクション構成（この順で実装）
1. **FV**：見出し/サブ/主CTA「14日間 無料ではじめる」/副CTA「機能を見る」/「インストール不要・カード登録は後でOK ※要確認」/スマホ実機モック。
2. **課題提起**：「こんな"手作業"、まだ続けていませんか？」＋チェックリスト（計算2-3時間／手数料ミス／シフトがLINEで埋もれる／予約バラバラ／バック何度も聞かれる）。
3. **ソリューション概要**：差別化3カード。
4. **機能/ベネフィット詳細**：機能名でなく"悩みの解決"を見出しに（委託金/シフト承認/本日ダッシュボード/接客メモ/PIN生体認証/緊急SOS/打刻・通知/マルチ店舗）。
5. **使い方3ステップ**：登録→コース・セラピスト登録→当日から管理。
6. **料金**：「まずは14日間無料」。月額（要確定）・いつでも解約・カード登録タイミング（要確認）。
7. **安心・セキュリティ**：PIN＋生体認証・権限管理・暗号化保管（※開発部と表現整合）。
8. **FAQ**（第7章）。
9. **最終CTA**：「月末の電卓を、今日で卒業しよう。」
10. **フッター**：特商法(tokutei.html)/プライバシー(legal.html)/問い合わせ。

## 5. CTA・誘導方針
- 文言統一：主「14日間 無料ではじめる」／短縮「無料で試す」／副「機能を見る」。
- FV・料金・最終の**3箇所に主CTA必須**＋スマホは**下部固定CTAバー**常時表示。
- フォームは**最小（メール＋パスワードのみ）**、「カードは今すぐ不要」を明記（※課金フローと整合確認）。

## 6. トーン&マナー・配色
- 誠実×親しみ×上質。煽らず現場に寄り添うやさしい日本語（専門用語回避＝「PWA」は「インストール不要」と言い換え）。品位を保つ。
- 配色：アクセント #FF9124／ベース白／文字は柔らかいダークグレー／淡オレンジ帯(#FFF3E8等)。余白広め・装飾過多にしない。実画面モックを主役に。

## 7. FAQ（7問・要旨）
1. PC苦手でも使える？→スマホが使えればOK、インストール不要。
2. DL必要？→不要（PWA）。
3. 委託金計算はどこまで自動？→手数料/10%上乗せ/日別月次/印刷・PDFまで。
4. セラピストも使える？→シフト申請・本日予約・売上確認、PIN＋生体認証。
5. 料金・無料期間→14日無料、月額（要実数）、いつでも解約。
6. 複数店舗→対応（マルチテナント）。
7. セキュリティ→PIN＋生体認証・権限管理・安全保管。

## 8. 確定待ち項目（経営/運用判断）
月額料金・解約条件・トライアル中のカード要否／導入実績数値（現状は載せない）／サポート体制／特商法の事業者住所（バーチャルオフィス審査後）／公開ドメイン／セキュリティ表記の正確性。

# 第2部　技術実装

## 0. 現状サマリ（是正対象）
- `lp.html` 既存・`vercel.json` で `/`→`/lp.html` 設定済み＝**ブラッシュアップが基本**。
- **アクセント色不一致**：lp.html `#F97316` vs legal/tokutei `#FF9124` → **#FF9124 に統一**。
- **ドメイン表記混在**：`shiftlink-app.jp` と `app.shiftlink.jp` → **1つに確定**して全リンク/OGP/canonical統一（デフォルトは現行稼働の shiftlink-app.jp）。
- OGP画像 `logo.png` の実体未配置。

## 1. 構成方針
- **単一HTML（lp.html）で完結**・ビルドなし・フレームワーク不使用。legal/tokutei と同流儀（headインライン`<style>`、Noto Sans JP）。
- 2,000行超で可読性が落ちる場合のみ `lp.css` 分離可。
- **index.html（アプリ本体）には一切触らない**。

## 2. ファイル構成・ルーティング
| 項目 | 仕様 |
|---|---|
| LP本体 | `/lp.html` |
| ルーティング | `vercel.json` rewrites `/`→`/lp.html`（設定済み） |
| アプリ | `/index.html`（PWA・触らない） |
| 規約/特商法 | `/legal.html`・`/tokutei.html`（既存） |
| 画像 | `/assets/lp/` に集約 |
| favicon/OGP | `/favicon.ico`・`/apple-touch-icon.png`・`/logo.png` を直下 |

導線：`/`→lp.html → 主CTA → `index.html?saas`（SaaSサインアップ）。フッター→legal/tokutei。

## 3. デザインシステム（:root・legal/tokuteiと統一）
`--ink:#211C16 / --sub:#7A6E5E / --mute:#A89B88 / --line:#EDE9E1 / --accent:#FF9124 / --accent-deep:#E67A12 / --bg:#FFFFFF / --soft:#FAF8F4 / --danger:#EF4444`
- フォント Noto Sans JP（400/500/700・display=swap）。
- モバイルファースト。BP 768/1024px。テキスト幅640-720px、グリッド最大960px中央寄せ。`overflow-x:hidden`。
- 共通クラス：`.lp-container/.btn-primary(タップ44px+)/.btn-ghost/.lp-card/.section-title/.section-desc`。白⇄`--soft`で交互背景。

## 4. セクション実装要件
各セクションを独立 `<section id>` で（既存ID `#pain/#features/#pricing/#faq` 基準）。ヘッダー追従(sticky)＋CTA／ヒーロー(H1は1つ・モック画像)／課題リスト／3ステップ(ol)／特徴グリッド(1→2-3列)／スクショ(`loading=lazy`・width/height・alt)／中間CTA／料金カード／FAQ／最終CTA／フッター。
- **アクセシビリティ**：見出し階層厳守(H1は1つ)、画像alt、color依存回避(コントラスト4.5:1+)、`button/a`使用・フォーカスリング維持。

## 5. CTA導線
- 主CTA→`https://<確定ドメイン>/index.html?saas`（クエリ表記を全所統一）。
- 全主CTA同一URL・文言。フッター：`./legal.html`・`./tokutei.html`・ログイン(?saas)・LINE(`https://lin.ee/9CREJEh`、`rel="noopener"`)。

## 6. SEO/OGP/メタ/パフォーマンス/計測
- title 60字・description 120字・canonical(確定ドメイン/)。JSON-LD `SoftwareApplication`(任意)。
- OGP一式＋**1200×630のOGP画像を実制作**、`twitter:card=summary_large_image`。
- favicon/apple-touch-icon追加。
- 画像WebP・`loading=lazy`・width/height明示(CLS防止)。Material IconsはインラインSVGへ置換推奨。
- 計測差し込み口：`<!-- ANALYTICS HEAD START -->`(/head直前)・`<!-- ANALYTICS BODY START -->`(body直後)。主CTAに `data-cta="..."`。

## 7. 画像・スクショ素材（/assets/lp/・2倍書き出し）
ヒーロー実機モック／委託金明細(3列・PDFプレビュー)／シフト承認・ルーム自動振り分け／予約管理／接客メモ・顧客管理／打刻・通知／OGP(1200×630)／favicon一式。
- **実個人情報は必ずダミー差し替え**（テスト店舗データ）。
