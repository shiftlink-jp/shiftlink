# リリース前チェックリスト

ShiftLinkの本番リリース（mainブランチへのマージ）前に、必ずこのチェックリストを実行する。
**1つでもFAILがあれば本番マージは中止し、開発部で修正してから再テストする。**

---

## 0. 事前準備

- [ ] stagingブランチに最新変更がpush済み
- [ ] VercelでstagingのプレビューURLが正常に表示される
- [ ] ブラウザのDevTools Consoleを開いた状態でテストする（エラー監視）

---

## 1. ログイン・認証（致命度：最高）

### オーナーPINログイン
- [ ] オーナーPINでログインできる
- [ ] 誤ったPINでログイン拒否される
- [ ] ログイン後にダッシュボード（本日タブ）が表示される
- [ ] 生体認証ボタンが表示される（登録済みの場合）

### キャストPINログイン
- [ ] キャスト一覧が表示される
- [ ] キャストを選択 → PIN入力 → ログインできる
- [ ] 誤ったPINでログイン拒否される
- [ ] ログイン後にキャスト用画面が表示される

### セッション復元
- [ ] ログイン状態でリロード → 自動でログイン状態が維持される
- [ ] ログアウト後にリロード → ログイン画面に戻る

---

## 2. 本日ページ（致命度：高）

- [ ] 本日のシフトが正しく表示される
- [ ] ルームグリッドが表示される（各ルームに割り当てられたキャストが見える）
- [ ] 問合せ数を入力・保存できる
- [ ] 日付ナビゲーション（前日・翌日・今日に戻る）が動く
- [ ] ガントモーダル（ルーム名タップ）が開く

---

## 3. 予約管理（致命度：高）

- [ ] 「予約追加」シートを開ける
- [ ] コース・オプション・指名タイプを選択できる
- [ ] 顧客番号入力 → 既存顧客の名前が自動補完される
- [ ] 新規顧客番号で予約登録できる（顧客テーブルにも自動追加される）
- [ ] 予約登録後、本日ページに反映される
- [ ] 予約編集できる
- [ ] 予約削除できる
- [ ] **担当キャストにPush通知が飛ぶ**

---

## 4. 委託金入力（致命度：高）

- [ ] 委託金シートが開く
- [ ] 予約データから自動でセッションが生成される
- [ ] コース・指名・オプション・延長の金額が正しく計算される
- [ ] 雑費・保証額が反映される
- [ ] 各セルをタップして手動修正できる（最新機能）
- [ ] 保存できる
- [ ] **キャストに「委託金が入力されました」通知が飛ぶ**
- [ ] 既に保存済みの場合は次回開いた時に保存内容が表示される

---

## 5. シフト管理（致命度：中）

### キャスト側
- [ ] シフトカレンダーが表示される
- [ ] 日付を選択してシフト申請できる
- [ ] 承認待ちシフトが表示される
- [ ] 既存シフトを訂正できる
- [ ] **オーナーに「シフトが提出されました」通知が飛ぶ**

### オーナー側
- [ ] シフト申請一覧が表示される
- [ ] 承認できる → **キャストに通知**
- [ ] 却下できる → **キャストに通知**

## 6. 顧客管理（致命度：中）

- [ ] 顧客一覧が表示される
- [ ] 顧客検索ができる
- [ ] 顧客情報を編集できる
- [ ] 来店履歴が表示される
- [ ] 顧客メモを保存できる

## 7. 管理画面（致命度：中）

- [ ] コース一覧の追加・編集・削除
- [ ] オプション一覧の追加・編集・削除
- [ ] ルーム一覧の追加・編集・削除（色設定含む）
- [ ] キャスト追加・編集・退店
- [ ] 委託金設定（cast_fees）が保存できる

- [ ] 店舗設定（営業時間・指名料）が保存できる

## 8. Push通知（致命度：最高）

- [ ] オーナーが通知設定ボタンから登録できる
- [ ] キャストが通知設定ボタンから登録できる
- [ ] 通知許可を一度解除した後でも再登録できる
- [ ] テスト予約を入れる → 担当キャストに通知が届く
- [ ] テスト委託金入力 → キャストに通知が届く
- [ ] **iPhone（PWAインストール済）で通知が届く**

---

## 9. レスポンス・パフォーマンス（致命度：低）

- [ ] 本日ページが2秒以内に表示される
- [ ] シート開閉のアニメーションがスムーズ
- [ ] スクロール中にタップが効かない事象がない
- [ ] スマホでキーボード表示時にフォームが隠れない

---

## 10. エラー監視（致命度：高）

- [ ] テスト中にブラウザDevTools Consoleにエラーが出ない
- [ ] テスト後にerror_logsテーブルを確認 → 想定外エラーがない

確認SQL：
```sql
SELECT created_at, user_name, error_type, error_message, url
FROM error_logs
WHERE created_at > now() - interval '1 hour'
ORDER BY created_at DESC;
```

---

## 11. SaaSモード（複数店舗提供前に必須）

`?saas=1` パラメータをつけて以下を確認：
- [ ] SaaSログインができる
- [ ] 自店舗のデータのみ表示される
- [ ] 他店舗のデータが一切混入しない
- [ ] INSERT時に自動で自店舗の `store_id` が付く

確認SQL（別店舗のデータが混入していないか）：
```sql
SELECT store_id, COUNT(*)
FROM reservations
WHERE created_at > now() - interval '1 day'
GROUP BY store_id;
```
